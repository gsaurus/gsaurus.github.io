Getting Started¶

Learn how to create your first script and understand the basics of the Lua integration.

Basic Structure¶

A Lua script is a text file with the .lua extension. The game executes these scripts when certain events occur (like an animation frame or a move update).

The entry point for a script is its main body. Most scripts return true to indicate success.

-- My first script
local character = game.this_character

if (character.is_hitting_something) then
    character.play_sound("Test chronicle", "impact", 1)
end

return true

The game Global¶

The game object is the main entry point to the engine. It provides access to the current frame, players, enemies, and items.

  • game.this_character: Returns the character currently executing the script (either a player or an enemy).
  • game.this_player: Returns the current player (if the script is running for a player).
  • game.this_enemy: Returns the current enemy (if the script is running for an enemy).

Object Hierarchy & Containment¶

In Streets of Rage New Era, game objects are built in layers. A high-level object like a Player is not just a player; it is a character that is also a physical item and an animatable sprite.

This is achieved through a "containment" model where each layer wraps the one below it:

  • Player or Enemy (The Logic Layer)
    • ⬇️ contains a...
  • Character (The Combat Layer)
    • ⬇️ contains an...
  • Item (The Physics Layer)
    • ⬇️ contains a...
  • SubItem (The Visual Layer)

Why does this matter?¶

Understanding this hierarchy helps you find where specific properties and methods are defined:

  1. SubItem (The Foundation): Handles the "visual" layer.
    • Animations, palette lines, sprite orientation (is_facing_right).
    • Basic sound playing.
  2. Item (The Physical Layer): Handles "physics" and world state.
    • Position (position_x, position_y), velocity, and impulses.
    • Statuses like is_vulnerable, is_knocked_down, or is_active.
  3. Character (The Entity Layer): Adds gameplay logic.
    • Health, lives, and display names.
    • Combat states like combo_count, victim, and attacker.
    • Methods for grabbing, hitting, and specialized movement.
  4. Player & Enemy (Specialized Logic):
    • Player: Adds input handling (is_holding_button_b, has_input_combo).
    • Enemy: Adds AI logic (mode, enemy_agressiveness, distance).

Accessing Properties (Flattening)¶

For your convenience, properties are "flattened". This means you can access properties from a lower layer (like Item or SubItem) directly on a higher one (like Player or Character) without having to go through a child property.

local player = game.this_player

-- These are all equivalent and perfectly valid:
player.position_x = 5000         -- Accessing Item property through Player (Flattened)
player.item.position_x = 5000    -- Accessing Item property explicitly
player.character.item.position_x = 5000 -- Full explicit path

Note: Properties like health are found at the Character level, while position_x is at the Item level. Because a Player contains a Character, you can just use player.health.

Entity IDs (Global Indices)¶

In the engine, every active object has a unique Entity ID (also referred to as object_index). This is a global integer that the engine uses to keep track of every player, enemy, item, and sub-item in the scene.

ID Ranges¶

The IDs are organized in contiguous ranges. While the exact starting numbers can vary depending on the game configuration, the order is always the same:

  1. Players: Usually the first IDs (e.g., 0 to 3).
  2. Enemies: Follow the players.
  3. Items: Follow the enemies (weapons, containers, etc.).
  4. Sub-Items: The final range of IDs.

Working with IDs vs. Objects in Lua¶

The API often provides two ways to access related entities: by ID (integer) or by Object (shortcut).

1. Using Objects (Direct Access)¶

Properties like character.attacker or character.victim are shortcuts. They automatically resolve the internal ID and return a full Lua object you can use immediately.

local character = game.this_character
local attacker = character.attacker -- Returns a Character object (or nil)

if (attacker ~= nil) then
    -- Direct access to properties
    if (attacker.health < 50) then
        character.play_sound("MyPack", "taunt", 0)
    end
end

2. Using IDs (Manual Resolution)¶

You can work with the Entity IDs directly, for example requesting the attacker_character_id and obtaining the respective character object using game.get_character(id) method.

local character = game.this_character
local attacker_id = character.attacker_character_id -- Returns an integer ID

if (attacker_id >= 0) then
    -- Convert the ID into a usable Character object
    local attacker = game.get_character(attacker_id)

    -- Now you can access its properties
    attacker.health = attacker.health - 10
end

Important Units¶

  • Positions: In-game positions are multiplied by 1000 for precision. When setting position_x, remember that 1 pixel is roughly 1000 units.
  • Animations: Animation IDs are divided by 2 in Lua (mapping back to the engine's internal representation). Just use the IDs as they appear in the Character Editor.
